# Debug OpenShell Cluster

Diagnose why a openshell cluster failed to start after `openshell gateway start`.

Use **only** `openshell` CLI commands (`openshell status`, `openshell doctor logs`, `openshell doctor exec`) to inspect and fix the cluster. Do **not** use raw `ssh` commands directly — always go through the `openshell doctor` interface. The CLI auto-resolves local vs remote gateways, so the same commands work everywhere.

## Overview

`openshell gateway start` starts a native gateway process using Apple Container for sandbox management.

The deployment stages are:

1. Detect Apple Container (`container` CLI on PATH).
2. Locate or build the `openshell-server` binary.
3. Generate PKI (mTLS certificates) at `~/.openshell/gateways/<name>/pki/`.
4. Start the gateway as a background process.
5. Write a PID file at `~/.openshell/gateways/<name>/gateway.pid`.
6. TCP health probe on the gateway's listen port until it accepts connections.

The default gateway name is `openshell`. The host port is configurable via `--port` (default 8080).

## Prerequisites

- Apple Container must be running
- The `openshell` CLI must be available
- For remote clusters: SSH access to the remote host

## Tools Available

All diagnostics go through `openshell` CLI commands:

```bash
# Quick connectivity check
openshell status

# Fetch gateway logs
openshell doctor logs --lines 100
openshell doctor logs --tail          # stream live
```

## Workflow

When the user asks to debug a cluster failure, **run diagnostics automatically** through the steps below in order. Stop and report findings as soon as a root cause is identified. Do not ask the user to choose which checks to run.

### Determine Context

Before running commands, establish:

1. **Gateway name**: Default is `openshell`
2. **Config directory**: `~/.openshell/gateways/{name}/`

### Step 0: Quick Connectivity Check

Run `openshell status` first. This immediately reveals:
- Which gateway and endpoint the CLI is targeting
- Whether the CLI can reach the server (mTLS handshake success/failure)
- The server version if connected

Common errors at this stage:
- **`tls handshake eof`**: The server isn't running or mTLS credentials are missing/mismatched
- **`connection refused`**: The container isn't running or port mapping is broken
- **`No gateway configured`**: No gateway has been deployed yet

### Step 1: Check Gateway Logs

Get recent gateway logs to identify startup failures:

```bash
openshell doctor logs --lines 100
```

Look for:

- Port binding failures
- Certificate/PKI errors
- Apple Container connectivity issues

### Step 2: Check mTLS / PKI

TLS certificates are generated by `openshell-bootstrap` and stored at `~/.openshell/gateways/<name>/pki/`.

```bash
# Check if CLI-side mTLS files exist locally
ls -la ~/.openshell/gateways/<name>/pki/
```

Common mTLS issues:
- **CLI can't connect after redeploy**: Check that the PKI directory contains `ca.crt`, `server.crt`, `server.key` and that they were updated at deploy time.
- **Local mTLS files missing**: The gateway was deployed but credentials weren't persisted (e.g., interrupted deploy). Destroy and recreate: `openshell gateway destroy && openshell gateway start`.

### Step 3: Check Apple Container

Verify Apple Container is running and functional:

```bash
container system info
```

If Apple Container is not running, start it:

```bash
container system start
```

## Common Failure Patterns

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `tls handshake eof` from `openshell status` | Server not running or mTLS credentials missing/mismatched | Check gateway PID file and mTLS files (Step 2) |
| `connection refused` | Gateway process not running or port conflict | Check if gateway PID is alive: `cat ~/.openshell/gateways/<name>/gateway.pid && ps -p $(cat ~/.openshell/gateways/<name>/gateway.pid)` |
| `No gateway configured` | No gateway has been deployed yet | Run `openshell gateway start` |
| Port conflict | Another service on the configured gateway host port (default 8080) | Stop conflicting service or use `--port` on `openshell gateway start` |
| Apple Container not running | `container system start` not run | Run `container system start` |
| Sandbox creation fails | Apple Container not running or vmnet issue | Check `container system info` and gateway logs |

## Full Diagnostic Dump

Run all diagnostics at once for a comprehensive report:

```bash
echo "=== Connectivity Check ==="
openshell status

echo "=== Gateway Logs (last 50 lines) ==="
openshell doctor logs --lines 50

echo "=== Apple Container Status ==="
container system info

echo "=== Gateway PID ==="
cat ~/.openshell/gateways/openshell/gateway.pid 2>/dev/null && ps -p $(cat ~/.openshell/gateways/openshell/gateway.pid 2>/dev/null) 2>/dev/null || echo "No PID file found"

echo "=== PKI Files ==="
ls -la ~/.openshell/gateways/openshell/pki/ 2>/dev/null || echo "No PKI directory"
```
